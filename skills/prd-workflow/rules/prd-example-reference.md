## 1. 프로젝트 개요

- **목표**: 파트너 관리자가 **웹 기반 콘텐츠 에디터**를 통해 콘텐츠를 직접 작성하고, 앱 홈 화면에 노출하여 사용자 참여도를 높인다.
- **기대효과**:
  - 콘텐츠 제작 리드타임 50% 단축 (외부 의존성 제거)
  - 홈 화면 체류시간 20% 증가 목표
  - 콘텐츠 클릭률 15% 목표
- **라이브 일정**: 2025-XX-XX (배포)

---

## 2. 사용자 시나리오 (Job Story)

> When **파트너 관리자가 새로운 콘텐츠를 제작해야 할 때**,
> I want to **웹 에디터에서 텍스트, 이미지, 유튜브를 쉽게 편집하고 앱에 즉시 반영하길 원하며**,
> So **별도의 개발 요청 없이 신속하게 사용자에게 유용한 정보를 전달할 수 있다**.

## 4. 디자인 & Flow

### 4.1 콘텐츠 에디터 홈 (목록 페이지)

- **화면 ID**: `EDITOR-HOME-001`
- **Figma 링크**: [홈\_콘텐츠 생성 후](https://www.figma.com/design/example/ID?node-id=1-3)

**[UI 요소 및 인터랙션]**

1. `Side Bar`: 좌측 사이드바에 "Editor" 로고 및 [콘텐츠 작성] 버튼 노출.
2. `Profile Area`: 우측 상단에 "관리자" 사용자명 + [로그아웃] 버튼.
3. `Table Header`: "타이틀", "최초 작성일", "앱 링크", "관리" 4개 컬럼.
4. `List Item`: 각 콘텐츠 행에 타이틀 텍스트, 작성일(YYYY.MM.DD), [링크 복사] 버튼, [관리] 버튼 배치.
5. `Hover State`: 행에 마우스 오버 시 배경색 #F5F5F5 적용.
6. `Empty State`: 콘텐츠 없을 시 빈 서랍 아이콘 + "아직 작성된 콘텐츠가 없습니다." 메시지 노출.

**[로직 및 정책 (Business Logic)]**

1. **권한 검증:**
   - 에디터 페이지 진입 시 PA 세션 토큰 확인.
   - 권한 없음 → 403 에러 페이지로 리다이렉트.

2. **목록 조회:**
   - `GET /api/v1/contents?page=1&limit=20` 호출.
   - 최신 작성일 기준 내림차순 정렬.

3. **링크 복사:**
   - [링크 복사] 버튼 클릭 시 `https://example.com/contents/{content_id}` 딥링크 생성.
   - 클립보드 복사 후 "링크가 복사되었습니다." 토스트 노출 (3초 자동 닫힘).

### 4.2 콘텐츠 작성/편집 페이지

- **화면 ID**: `EDITOR-WRITE-001`
- **Figma 링크**: [에디터 옵션](https://www.figma.com/design/example/ID?node-id=1-4)

**[UI 요소 및 인터랙션]**

1. `Editor Toolbar`: 상단에 [제목], [본문], [볼드], [이탤릭], [색상], [이미지 추가], [유튜브 추가] 버튼 배치.
2. `Title Input`: 단일 라인 텍스트 입력, 최대 50자.
3. `Content Editor`: 멀티라인 리치 텍스트 에디터.
4. `Image Upload`: [이미지 추가] 클릭 → 파일 선택 모달 → 업로드 진행률 표시 → 에디터 내 이미지 삽입.
5. `YouTube Embed`: [유튜브 추가] 클릭 → URL 입력 모달 → 유효성 검사 후 플레이어 임베드.
6. `Thumbnail Upload`: 하단에 썸네일 이미지 업로드 영역 (필수, 최소 600x400px 권장).
7. `Save Button`: 우측 하단 [저장하기] 버튼, 클릭 시 로딩 스피너 노출.

**[로직 및 정책 (Business Logic)]**

1. **유효성 검사:**
   - 타이틀 미입력 시: "타이틀을 입력해주세요." 에러 메시지.
   - 본문 미입력 시: "본문을 입력해주세요." 에러 메시지.
   - 썸네일 미업로드 시: "썸네일을 업로드해주세요." 에러 메시지.

2. **이미지 업로드:**
   - `POST /api/v1/upload/image` 호출.
   - 최대 10MB, jpg/png 형식만 허용.
   - 업로드 실패 시: "이미지 업로드에 실패했습니다. 다시 시도해주세요." 토스트 노출.

3. **유튜브 URL 검증:**
   - Regex: `^(https?\:\/\/)?(www\.youtube\.com|youtu\.?be)\/.+$`
   - 유효하지 않을 시: "올바른 유튜브 URL을 입력해주세요." 에러 메시지.

4. **저장 처리:**
   - `POST /api/v1/contents` 호출.
   - **Success (200):** "콘텐츠가 저장되었습니다." 토스트 노출 → 목록 페이지로 이동.
   - **Fail (400):** 에러 메시지 파싱하여 사용자에게 표시.
   - **Fail (500):** "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요." 토스트 노출.

### 4.3 앱 홈 - 콘텐츠 영역

- **화면 ID**: `APP-HOME-CONTENTS-001`

**[UI 요소 및 인터랙션]**

1. `Section Title`: "추천 콘텐츠" 텍스트 + [전체 보기] 버튼.
2. `Content Cards`: 가로 스크롤 가능한 카드 리스트 (썸네일 + 타이틀).
3. `Card Interaction`: 카드 클릭 시 콘텐츠 상세 페이지로 이동.
4. `[전체 보기] Button`: 클릭 시 콘텐츠 목록 페이지로 이동.

**[로직 및 정책 (Business Logic)]**

1. **콘텐츠 조회:**
   - `GET /api/v1/app/contents?status=published&limit=10` 호출.
   - PA에서 노출 설정된 콘텐츠만 반환.
   - 노출 시작일 기준으로 최신순 정렬.

2. **노출 위치:**
   - 홈 화면에서 "추천 섹션"과 "배너(DA)" 사이에 삽입.
   - 뷰포트에 100% 노출 시 `impression_home_contents` 이벤트 발생.

3. **스와이프 동작:**
   - 사용자가 좌우 스와이프하여 다음 콘텐츠 노출 시 새로운 콘텐츠에 대해 `impression_home_contents` 이벤트 추가 발생.

### 4.4 앱 콘텐츠 상세 페이지

- **화면 ID**: `APP-CONTENTS-DETAIL-001`

**[UI 요소 및 인터랙션]**

1. `Header`: 좌측 [< 뒤로가기] 버튼, 중앙 빈 공간, 우측 [공유하기] 아이콘.
2. `Content Body`: 타이틀(H1), 작성자("서비스 기획팀"), 작성일(YYYY.MM.DD), 본문(리치 텍스트), 이미지, 유튜브 영상.
3. `Like Button`: 하단 고정 영역에 하트 아이콘 + 좋아요 수 표시.
4. `Share Button`: [공유하기] 아이콘 클릭 시 네이티브 공유 바텀시트 노출.

**[로직 및 정책 (Business Logic)]**

1. **콘텐츠 조회:**
   - `GET /api/v1/app/contents/{content_id}` 호출.
   - 콘텐츠 없음(404) → "콘텐츠를 찾을 수 없습니다." 에러 페이지 노출.

2. **좋아요 토글:**
   - 하트 아이콘 클릭 시 `POST /api/v1/app/contents/{content_id}/like` 호출.
   - 좋아요 상태: 하트 색상 #FF4081, 카운트 +1.
   - 해제 상태: 하트 색상 #CCCCCC, 카운트 -1.

3. **공유하기:**
   - 네이티브 공유 바텀시트에서 "카카오톡" 선택 시:
     - 썸네일 이미지 URL + 타이틀 + 본문 첫 100자 + [자세히보기] 버튼(딥링크) 전달.
     - 오픈그래프 메타태그: `og:title`, `og:description`, `og:image`, `og:url` 설정.

4. **뒤로가기:**
   - [< 뒤로가기] 클릭 시 콘텐츠 목록 페이지(`APP-HOME-CONTENTS-LIST-001`)로 이동.

  - 권한 관리 API 연동 필요 (FE-1003).
  - 앱 내 콘텐츠 영역 개발(FE-1002) 완료 후 통합 테스트 진행.
  - QA 완료(QA-1001) 후 배포 가능.
